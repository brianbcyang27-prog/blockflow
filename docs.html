BlockFlow

A productivity app to reduce scrolling addiction and improve focus

In Development

📌 Project Goals

The Problem

問題現況

  • 現代人每天花費大量時間滑手機
  • 美國成年人平均每天解鎖手機 80-150 次
  • 每次分心後需要約 23 分鐘才能回覆專注
  • 大學生的平均睡眠不足 7 小時
  • 缺乏簡單的時間管理工具

解決方案

  • 採用「3 區塊日」時間管理法
  • 结合 Pomodoro 計時器追蹤專注
  • 記錄分心次數提高自覺
  • 追蹤睡眠養成健康習慣
  • 簡單易用的 Web 介面

User Stories

作為一個大學生
我希望記錄每天的專注/休閒時間
以平衡學業和生活
作為一個容易分心的人
我想追蹤自己滑手機的次數
以提高自覺並改善習慣
作為一個注重健康的人
我想記錄睡眠時間
以了解自己的睡眠質量

⚙️ Features

🎯

3 個可自訂時長的日常區塊

採用「3 區塊日」概念，將每天分成三大類別：

Focus Block - 工作、學習、學校（寫作業、準備考試）
Personal Block - 興趣、專案（學吉他、寫小說）
Recovery Block - 休息、社交、娛樂（看劇、聊天）
每個區塊可設定 15-240 分鐘，完成後可標記為已完成。
⏱️

Pomodoro 風格計時器

25 分鐘番茄工作法定時器，幫助用戶保持專注

使用方法： 點擊「Start Focus」開始 25 分鐘倒數，計時結束後記錄專注時間。用戶可隨時暫停或重置。
🚫

分心次數追蹤

當用戶發現自己在滑手機時，按下「I got distracted」記錄一次

目的： 提高自覺，了解自己一天分心多少次。
參考： 0-3次=專注，4-7次=普通，8次以上=需改善
😴

睡眠時間記錄

設定就寢時間和起床時間，自動計算睡眠時長

使用情境： 睡前設定 Sleep Time，起床後設定 Wake Time，系統自動計算。
評價： <7小時=不足，7-8小時=理想，>8小時=可能過多
🔄

每日自動重置

每天午夜自動將資料歸零，開始新的一天

設計理念： 每天早上是新的開始，前一天的未完成不應成為壓力。
歷史記錄： 過去 7 天的資料會被保存用於週統計。
🔐

Google 登入（Firebase Auth）

使用 Google 帳號登入，未來可同步資料到雲端

目前狀態： 已完成 Firebase Authentication 整合
目的： 未來可連接 Google Calendar 讀取行事曆事件
📅

Google Calendar 事件顯示

在月曆上顯示 Google Calendar 的行程

使用情境： 用戶可在同一介面看到「計畫」和「行事曆」，更好規劃時間。
目前狀態： Firebase 部署遇到困難，功能仍在開發中

Challenges

⚠️ Firebase Cloud Functions 部署困難

Firebase Cloud Functions 需要升級到 Blaze 方案才能部署。Blaze 方案需要信用卡驗證，雖然有每月免費額度（125,000 次調用），但需要驗證才能使用。

影響： Google Calendar 事件顯示功能暫時無法正常運作

解決方案： 1) 升級 Firebase 方案 2) 使用其他後端服務 3) 手動輸入行事曆（未來優化）
